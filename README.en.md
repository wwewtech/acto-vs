## Why You Need This

When working with large language models (ChatGPT, Claude, Gemini...), you often need to provide them with your code context. Copying files manually is slow and error-prone.

**ACTO** solves this: select the files you need in the project tree, press one button — and get a ready-made report with a structure tree and full contents of all selected files.

---

## ✨ Features

| | |
|---|---|
| 📂 **File Tree** | Convenient sidebar panel with checkboxes for every file and folder |
| ☑️ **Cascading Selection** | Clicking a folder instantly selects all nested files |
| ⚡ **Select All / Deselect All** | Toolbar buttons for bulk selection |
| 🚫 **Smart Filtering** | Auto-skips `node_modules`, `.git`, `dist`, binary files |
| 🛑 **Stop on the Fly** | Interrupt collection at any time — partial results are saved |
| 📊 **Token Counting** | Token estimate shown directly in the status bar |
| 📋 **Copy to Clipboard** | One click — context is already in your clipboard |
| 🗂️ **ASCII Tree** | Report includes a visual file structure of the project |

## 🚀 Quick Start

### Installation

**Via Marketplace:**

1. Open VS Code
2. Go to Extensions (`Ctrl+Shift+X`)
3. Search for **ACTO** and click Install

**Manual (VSIX):**

```bash
code --install-extension acto-1.0.0.vsix
```

### Usage

```
1. Open your project folder in VS Code
2. Click the ACTO icon in the Activity Bar (left sidebar)
3. Check the files and/or folders you need
4. Click ▶ (Collect Report)
5. output.txt opens automatically — copy the contents to your LLM
```

## ⌨️ Commands

| Command | Shortcut | Description |
|---|---|---|
| `ACTO: Parse Selected Files` | — | Runs collection and creates `output.txt` |
| `ACTO: Copy to Clipboard` | — | Copies result without saving a file |
| `Select All` | — | Marks all project files |
| `Deselect All` | — | Removes all marks |
| `Stop Parsing` | — | Interrupts current collection |
| `Refresh Tree` | — | Re-reads the project structure |
| `ACTO: Settings` | — | Opens the settings page |

---

## ⚙️ Settings

| Parameter | Default | Description |
|---|---|---|
| `acto.ignoredFolders` | `["node_modules", ".git", "dist", ...]` | Folders that are always skipped |
| `acto.outputFileName` | `"output.txt"` | Result file name |
| `acto.maxFileSizeKb` | `512` | Maximum size of a single file (KB) |

## 🏗️ Development

```bash
# Clone the repository
git clone https://github.com/acto-dev/acto-vs.git
cd acto-vs

# Install dependencies
npm install

# Run in development mode
npm run watch

# Run tests
npm test
```

The extension is written in **TypeScript** and bundled with **esbuild**.  
For debugging, open the project in VS Code and press `F5`.
